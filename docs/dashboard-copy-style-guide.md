# Dashboard Copy Style Guide (RU+EN)

## Tone
- Professional, concise, and neutral.
- No guaranteed-return language.
- Clear cause -> action -> outcome phrasing.

## Message Patterns
- **Status:** short noun phrase (`Pending review`, `Completed`, `Rejected`).
- **Error:** what failed + next step (`Could not load withdrawals. Try again.`).
- **Success:** completed action in past tense (`Withdrawal request submitted.`).
- **Confirm:** explicit consequence + choice (`Cancel this request? This action cannot be undone.`).
- **Disclosure:** compliance/safety framing without fear tone.

## CTA Rules
- Start with a verb: `Confirm`, `Retry`, `Submit`, `Close`, `Open`.
- Keep under 3 words when possible.

## Glossary RU <-> EN
- `Заявка на вывод` <-> `Withdrawal request`
- `На проверке` <-> `In review`
- `Отклонено` <-> `Rejected`
- `Завершено` <-> `Completed`
- `Сбой` <-> `Failed`
- `Повторить` <-> `Retry`
- `Подтвердить` <-> `Confirm`
- `Риск необратимой операции` <-> `Irreversible transaction risk`
- `Проверка безопасности` <-> `Security review`
- `Проверка комплаенса` <-> `Compliance review`

## Forbidden Wording
- Any fixed ROI / guaranteed profit claims.
- Vague prompts without next step (`Something went wrong`).
- Mixed informal/formal register in one flow.
